# Contributor Guidelines

We welcome contributions to snacks-bibtex.nvim. Please follow these steps when proposing changes:

1. Discuss major changes in an issue or pull request draft before implementing them when possible.
2. Create a feature branch for your work and keep commits focused.
3. Follow the repository's [Agent Guidelines](AGENTS.md) for documentation, verification, and coding standards.
4. Ensure every modified, added, or removed function is documented inline (e.g., LuaDoc/EmmyLua annotations) and reflected in user-facing docs such as the README when applicable.
5. Run available verification commands (see `AGENTS.md`) and note the results in your pull request.
6. Provide a clear summary of your changes, affected modules, and any new configuration options in the pull request description.

Thank you for helping improve snacks-bibtex.nvim!
